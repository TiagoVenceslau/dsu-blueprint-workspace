# constant DSUBlueprint

DSU Blueprint Decorator

Defines a class as a DSU Blueprint, enabling:

  • Automatic CRUD operations, just by updating its corresponding DSUModel instance and running it through the appropriate OpenDSURepository;
  • Automatic validations && easily extendable for added validations;
  • Automatic serialization -> transmission -> deserialization;
  • Controlled accesses: Ability easily to add business logic at key points of any CRUD operations

Supported KeySSITypes:

  • KeySSIType.SEED: Expects:
    • keySsiType: KeySSIType.SEED;
    • specificKeyArgs: SeedSSISpecificArgs | undefined;
    • props: none
  • KeySSIType.ARRAY: Expects:
    • keySsiType: KeySSIType.ARRAY;
    • specificKeyArgs: ArraySSISpecificArgs | undefined;
    • props: {string[] | undefined} array of property names in the value chain notation getValueFromValueChain createObjectToValueChain that will be resolved from the DSUModel's content and used to pass as KeySSI generation Arguments.

DSU Blueprint Decorator

Defines a class as a DSU Blueprint, enabling:

  • Automatic CRUD operations, just by updating its corresponding DSUModel instance and running it through the appropriate OpenDSURepository;
  • Automatic validations && easily extendable for added validations;
  • Automatic serialization -> transmission -> deserialization;
  • Controlled accesses: Ability easily to add business logic at key points of any CRUD operations

Supported KeySSITypes:

  • KeySSIType.SEED: Expects:
    • keySsiType: KeySSIType.SEED;
    • specificKeyArgs: SeedSSISpecificArgs | undefined;
    • props: none
  • KeySSIType.ARRAY: Expects:
    • keySsiType: KeySSIType.ARRAY;
    • specificKeyArgs: ArraySSISpecificArgs | undefined;
    • props: {string[] | undefined} array of property names in the value chain notation getValueFromValueChain createObjectToValueChain that will be resolved from the DSUModel's content and used to pass as KeySSI generation Arguments. Must resolve to strings
    • extra Key generation args {string[] | undefined} that are passed to the OpenDSURepository's methods will be appended to the ones from the props and also used to generate the KeySSI
  • KeySSIType.WALLET: Expects:
    • keySsiType: KeySSIType.WALLET;
    • specificKeyArgs: WalletSSISpecificArgs | undefined;
    • props: {string[] | undefined}: array of property names in the value chain notation getValueFromValueChain createObjectToValueChain that will be resolved from the DSUModel's content and used to pass as KeySSI generation Arguments. Must resolve to strings
    • extra Key generation args {string[] | undefined} that are passed to the OpenDSURepository's methods will be appended to the ones from the props and also used to generate the KeySSI
    • One Supported decorator under the ConstantsApi#CODE_FOLDER property key (defaults to 'code'). Supported decorators are:
      • fromWeb;
      • fromCache;
      • mount

OpenDSURepository OperationKeys's this decorator acts on:

  • OperationKeys.CREATE: Creates the DSUBlueprint:
    • Gets the keySSI and the DSUFactoryMethod's DSUAnchoringOptions from the KeySSIFactoryRegistry;
    • Creates the DSU via the DSUFactoryRegistry#build
  • OperationKeys.READ:
  • OperationKeys.UPDATE:
  • OperationKeys.DELETE:

DSUOperationPhase phases this decorator acts on:

  • DSUOperationPhase.CLASS:

# fromWeb(appOrUrl, slot, triggerMountopt, deriveopt, mountPathopt, mountOptionsopt, keyOverrideopt, argsopt)

Depending on whether and url or an app name is supplied:

  • App Name: Communicates with ApiHub to get seeds from SSApps installed in that ApiHub instance:
    • slot 'primary': gets the seed file from the App's 'wallet-patch' folder;
    • slot 'secondary' gets the seed file from the App's ConstantsApi#APP_FOLDER

Retrieves the Seed during the DSUOperationPhase.CREATION operation and DBOperations.CREATE phases and stores in the in object under the matching property key

calls mount on this property to mount the seed during the DSUOperationPhase.EDITING operation;
